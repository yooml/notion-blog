Storage overhead

When objects are stored on disk, they are serialized to byte array using a compact, binary encoding. There is overhead added on top of serialied object data:

  • 8 bytes for object header (to record object size and oid, both 32-bit integers)
  • the size is rounded up to the 32 bytes (i.e. object of size between 0 and 32 bytes will take 32 bytes)
There's also less significant overhead for maintaining database structure.
